Production Case Studies

Smart Commute

Lead Engineer — Final Year Capstone Project

Flutter • Dart • Cloud Firestore

The Challenge

Inter-city daily commuters rely on informal, insecure communication chains. There is no centralized, real-time mechanism to match empty seats with traveling passengers securely.

The Architectural Choice

Chose a NoSQL document hierarchy using Firestore reactive streams rather than traditional polling architecture, establishing live, sub-second UI updates across active routes.

The Hardest Bug

Resolved race conditions during multi-passenger seat booking by implementing atomic Firestore transaction operations, locking down coordinate accuracy across mutations.

SyncPulse

Resilient Offline-First Synchronization Infrastructure

Flutter • Riverpod • Hive DB • Python

The Challenge

Mobile applications frequently drop data or duplicate state mutations when users transition through patchy cellular zones, creating severe data inconsistencies.

The Architectural Choice

Engineered an offline-first architecture utilizing Hive DB for localized request queuing, paired with Riverpod for immutable state propagation across the client UI.

Serverless Resolution

Integrated Python-based Firebase Cloud Functions to process the incoming queue, acting as a transactional authority to resolve synchronization conflicts upon network recovery.
